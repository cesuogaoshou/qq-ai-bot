# 产品规格书

## 产品定位

本项目是一个 QQ 群混合型大模型机器人，定位为“轻量群友 + 可控助手”。

它应该像群友一样自然，但不能抢话；在被 @ 或被明确请求时，又能像助手一样认真回答、总结上下文、辅助讨论。管理员必须始终拥有最高控制权，可以关闭机器人、查看状态、限制频率和清理上下文。

## MVP 目标

MVP 只服务一个固定 QQ 群，用来验证三件事：

1. 技术闭环是否稳定：能接收群消息、调用大模型、发送群消息。
2. 群聊体验是否可接受：不会频繁插话，不会长篇大论，不会拱火。
3. 管理控制是否可靠：管理员能随时关闭、限频和查看运行状态。

MVP 的一句话目标：

```text
在一个小 QQ 群内，机器人能稳定接收消息；仅在 @、管理员命令等明确场景下回复；回复短、可控、可关闭；失败能查日志。
```

## MVP 范围

MVP 必做能力：

1. 接入一个固定 QQ 群。
2. 接收群消息。
3. 识别 @ 机器人消息。
4. @ 触发大模型回复。
5. 保存最近 30 条左右群消息作为短期上下文。
6. 支持管理员命令：`/bot on`、`/bot off`、`/bot status`。
7. 支持默认人设。
8. 支持每群限频和每用户限频。
9. 使用 SQLite 保存配置、消息和限频状态。
10. 记录关键运行日志。

MVP 暂不做：

- 关键词主动触发，默认关闭，后续再灰度。
- 冷场接话，后续再灰度。
- 群级人设在线编辑。
- 中文命令别名。
- 图片理解、语音识别、表情包回复。
- Web 管理后台。
- 多平台接入。
- 多群配置管理。
- 长期复杂用户画像。
- 自动踢人、禁言等群管理动作。

## 后续可选能力

MVP 稳定后可以逐步加入：

- 关键词触发，例如“机器人”“帮我想想”“总结一下”。
- 冷场接话，但必须满足低频、低概率和上下文相关。
- `/bot mode mention-only|quiet|active`。
- `/bot memory clear`。
- `/bot persona 查看` 和 `/bot persona 设置 <文本>`。
- 多群配置，每个群独立开关、模式、人设和限频。
- 群聊日报、周报。
- 多模型切换。

## 交互策略

### 回复优先级

从高到低：

1. 管理员命令：必须处理。
2. 被 @：通常回复。
3. 明确关键词：后续阶段可启用。
4. 群聊冷场：后续阶段可启用。
5. 普通闲聊：默认不回复。

### 不回复条件

以下场景默认不回复：

- 群里正在连续高速聊天。
- 上一次机器人发言刚过去不久。
- 消息只是表情、短字或无明确语义。
- 用户在争吵中要求机器人站队。
- 内容涉及隐私、攻击、违法或高风险指导。
- 机器人已被管理员关闭。
- 命中群限频或用户限频。

## 默认人设

```text
你是这个 QQ 群里的轻量群友型助手。
你的目标是让群聊更自然、更舒服，而不是抢话。
平时回复要短，像普通群友。
被 @ 或被请求帮忙时，可以认真回答，但不要写得太长。
你可以轻微吐槽和接梗，但不能人身攻击。
群里争吵时优先降温。
不确定群内背景时先问一句，不要装懂。
不要频繁说“我是 AI”。
```

## 管理员命令

MVP 支持：

```text
/bot on
/bot off
/bot status
```

后续支持：

```text
/bot mode mention-only
/bot mode quiet
/bot mode active
/bot memory clear
/bot persona 查看
/bot persona 设置 <文本>
```

中文别名放到后续阶段，例如：

```text
/开启机器人
/关闭机器人
/查看状态
/清空记忆
```

## 成功标准

MVP 成功标准：

- 能稳定接收目标群消息。
- 被 @ 后能回复。
- 普通消息默认不触发回复。
- 回复长度受控，不刷屏。
- 管理员能开启、关闭和查看状态。
- 限频生效。
- 大模型超时时不崩溃。
- OneBot 发送失败时有日志。
- 连续运行数小时不崩溃。

## 风险

主要风险：

- QQ 协议端是非官方接入，存在账号风控风险。
- 大模型机器人容易变得吵、长篇大论或抢话。
- 用户可能诱导机器人输出攻击、隐私、违法或高风险内容。
- 群消息上下文可能包含隐私内容，存储必须克制。

应对原则：

- 使用小号和小群灰度。
- 默认 `mention_only`。
- 限制回复长度和频率。
- 管理员必须能一键关闭。
- 不做敏感属性推断，不做长期复杂用户画像。
