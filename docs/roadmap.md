# Roadmap

## 阶段 0：方案确认

目标：

- 明确产品定位。
- 确认 MVP 只支持一个固定群。
- 明确技术路线：OneBot、Python、SQLite、远程大模型 API。
- 确认第一版不使用 Docker。

交付物：

- README。
- 产品规格书。
- 技术选型设计。
- 架构设计。
- 开发规约。

## M1：本地消息闭环

状态：已完成代码实现，已完成真实群聊联调。

目标：

- 能连接 OneBot。
- 能接收目标群消息。
- 能发送一条固定回复。
- 能忽略非目标群消息。

验收标准：

- 启动后能稳定连接协议端。
- 收到目标群消息时有日志。
- 测试命令或固定触发能发送群消息。
- 连接失败和发送失败有错误日志。

## M2：@ 触发大模型回复

状态：已完成基础代码实现，进入本地联调和稳定化。

目标：

- 识别 @ 机器人。
- 读取最近上下文。
- 调用大模型 API。
- 将回复发回群。

验收标准：

- 普通消息默认不回复。
- @ 消息触发回复。
- 回复包含最近上下文影响，但不会泄露无关信息。
- 大模型超时不会导致进程崩溃。
- 回复长度受控。

## M2.5：搜索、安全和成本策略

状态：基础策略层完成，真实搜索供应商待选型接入。

目标：

- 建立联网搜索工具接口，默认关闭。
- 建立搜索触发策略，只在明确时效性问题或显式要求时触发。
- 建立安全分级策略，避免过度拒答。
- 建立高级能力成本配置和调用日志。

验收标准：

- 普通聊天不会自动搜索。
- 显式“搜一下/查一下”可以触发搜索流程。
- 搜索、图片、语音都有独立开关。
- 安全策略能区分正常问题、争吵、隐私和高风险请求。
- 无 API key 或工具关闭时，主聊天链路不崩溃。

## M3：可控上线

状态：基础代码实现完成，进入本地联调和稳定化。

目标：

- 支持管理员命令。
- 支持开关。
- 支持状态查看。
- 支持群限频和用户限频。
- 支持 SQLite 持久化。

验收标准：

- `/bot on` 可以开启机器人。
- `/bot off` 可以关闭机器人。
- `/bot status` 可以查看开关、模式和限频状态。
- 限频命中时不调用大模型。
- 重启后基础配置仍然存在。

## M4：聊天总结和记忆

状态：基础代码实现完成，进入本地联调和稳定化。

目标：

- SQLite 消息存储。
- 最近聊天总结。
- 群级长期摘要。
- 用户级轻量偏好记忆。
- 记忆查看和清理命令。

验收标准：

- 能总结最近聊天，不暴露个人隐私。
- 能形成群级摘要。
- 用户级记忆只记录非敏感偏好和互动风格。
- 记忆可查看、可删除。

## M5：图片和多模态

目标：

- 读图。
- 图片输出。
- 图片能力限频和成本控制。

验收标准：

- 用户明确要求时才处理图片。
- 图片内容默认不长期存储。
- 图片能力可关闭、可限频。

## M6：语音能力评估

目标：

- 评估协议端语音文件获取。
- 评估转码和语音转文字服务。
- 判断是否值得进入实现。

验收标准：

- 明确协议端是否能稳定获取语音文件。
- 明确语音转文字成本。
- 成本或稳定性不合适时允许放弃。

## M7：多群配置

目标：

- 从单群扩展到多个群。
- 每个群独立配置开关、模式、人设和限频。
- 不同群之间上下文隔离。

验收标准：

- 一个群关闭不影响其他群。
- 每个群使用独立上下文。
- 每个群有独立限频。
- 管理员权限按群隔离。

## 长期方向

长期可考虑：

- Web 配置页。
- Docker 或服务化部署。
- 日志监控和自动重启。
- 群聊日报、周报。
- 图片理解。
- 表情包回复。
- 多模型切换。
- 更精细的记忆管理。
- 图片生成。
- 语音转文字。

长期仍不建议默认开启：

- 自动踢人或禁言。
- 高频主动发言。
- 无边界用户画像。
- 多大群同时高强度运行。
