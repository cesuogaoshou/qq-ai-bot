# Roadmap

## 阶段 0：方案确认

目标：

- 明确产品定位。
- 确认 MVP 只支持一个固定群。
- 明确技术路线：OneBot、Python、SQLite、远程大模型 API。
- 确认第一版不使用 Docker。

交付物：

- README。
- 产品规格书。
- 技术选型设计。
- 架构设计。
- 开发规约。

## M1：本地消息闭环

状态：已完成代码实现，已完成真实群聊联调。

目标：

- 能连接 OneBot。
- 能接收目标群消息。
- 能发送一条固定回复。
- 能忽略非目标群消息。

验收标准：

- 启动后能稳定连接协议端。
- 收到目标群消息时有日志。
- 测试命令或固定触发能发送群消息。
- 连接失败和发送失败有错误日志。

## M2：@ 触发大模型回复

状态：已完成基础代码实现，进入本地联调和稳定化。

目标：

- 识别 @ 机器人。
- 读取最近上下文。
- 调用大模型 API。
- 将回复发回群。

验收标准：

- 普通消息默认不回复。
- @ 消息触发回复。
- 回复包含最近上下文影响，但不会泄露无关信息。
- 大模型超时不会导致进程崩溃。
- 回复长度受控。

## M3：可控上线

目标：

- 支持管理员命令。
- 支持开关。
- 支持状态查看。
- 支持群限频和用户限频。
- 支持 SQLite 持久化。

验收标准：

- `/bot on` 可以开启机器人。
- `/bot off` 可以关闭机器人。
- `/bot status` 可以查看开关、模式和限频状态。
- 限频命中时不调用大模型。
- 重启后基础配置仍然存在。

## M4：小群灰度

目标：

- 在一个小群中低频运行。
- 调整回复风格。
- 观察账号风控、延迟、成本和群友反馈。

验收标准：

- 连续运行数小时不崩溃。
- 群友不会明显觉得机器人刷屏。
- 日志能定位常见失败原因。
- 管理员能快速停用。

## M5：体验增强

可选能力：

- 关键词触发。
- 冷场接话。
- `/bot memory clear`。
- `/bot mode mention-only|quiet|active`。
- 群级人设查看和设置。
- 中文命令别名。

原则：

- 默认仍保持克制。
- 新触发能力先默认关闭。
- 每个能力都要有独立开关和限频。

## M6：多群配置

目标：

- 从单群扩展到多个群。
- 每个群独立配置开关、模式、人设和限频。
- 不同群之间上下文隔离。

验收标准：

- 一个群关闭不影响其他群。
- 每个群使用独立上下文。
- 每个群有独立限频。
- 管理员权限按群隔离。

## 长期方向

长期可考虑：

- Web 配置页。
- Docker 或服务化部署。
- 日志监控和自动重启。
- 群聊日报、周报。
- 图片理解。
- 表情包回复。
- 多模型切换。
- 更精细的记忆管理。

长期仍不建议默认开启：

- 自动踢人或禁言。
- 高频主动发言。
- 无边界用户画像。
- 多大群同时高强度运行。
